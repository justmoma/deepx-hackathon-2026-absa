The Netlify deploy errored, with the following guidance provided:

Diagnosis

- Relevant log lines showing large wheel downloads: [lines 193–201](#L193-L201).  
- The immediate error is "No space left on device" while pip is writing temporary files: [line 210](#L210), [line 226](#L226) and the resulting pip ProtocolError: [line 287](#L287).  
- Netlify fails the build during dependency installation as shown here: [lines 288–290](#L288-L290).

Root cause: pip is downloading very large PyTorch / CUDA-related wheels during pip install (see the large nvidia/torch wheel downloads at [lines 193–201](#L193-L201)). The build VM runs out of disk space while pip writes temporary files, causing OSError 28 and aborting dependency installation.

Solution

Short-term (fast, minimal changes)
1. Prevent pip from caching wheel files (reduces disk usage during install). In your Netlify site settings add the environment variable:
   - Key: PIP_NO_CACHE_DIR
   - Value: true

   Or set it in your build command, e.g.
```bash
export PIP_NO_CACHE_DIR=true && pip install -r requirements.txt
```

2. Also disable pip version checks to avoid extra writes:
   - Add env var: PIP_DISABLE_PIP_VERSION_CHECK=true

Medium / recommended (fix the root cause)
1. Remove heavy ML packages (torch, large CUDA wheels) from the Netlify build. Netlify is intended for static/front-end and small backends and has limited disk/compute. If your app needs to run ML inference or install PyTorch/transformers, host that service elsewhere (e.g., a dedicated server, cloud VM, API hosted on Render/Heroku/GCP/AWS/Lambda, or a container built with GitHub Actions) and let the Netlify site call that API.

2. If you must install PyTorch on Netlify (not recommended), switch to the CPU-only wheel (much smaller) and point pip to the PyTorch CPU index. Edit requirements.txt (see current file in your repo: https://github.com/justmoma/deepx-hackathon-2026-absa/blob/main/requirements.txt) and replace the torch entry with CPU wheel instructions. Example requirements.txt snippet:
```text
--extra-index-url https://download.pytorch.org/whl/cpu
torch==2.10.0+cpu
transformers==5.0.0
```
Notes:
- Verify the package line is present in requirements.txt (you already have torch==2.10.0 there) before changing it.
- Using the CPU index will avoid pulling CUDA (nvidia) wheels that consume lots of space.

Long-term / best practice
- Split responsibilities: keep the Netlify site for frontend/static assets, and deploy a separate backend (container or server) for heavy ML dependencies. Build your backend in an environment suited to large Python packages and GPU/CPU choices.
- If you choose to build artifacts (wheels, model files) in CI, build them in GitHub Actions or another CI that has larger disk and then deploy only the compiled artifact to Netlify (or host the model elsewhere and fetch at runtime).

Why these changes
- The log shows pip ran out of space while writing temp files during large downloads ([lines 193–201] and [line 210]), so reducing what pip downloads (CPU-only wheel) or disabling caching reduces disk writes and prevents the OSError 28 failure.
